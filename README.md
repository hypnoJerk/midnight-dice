# Midnight: 1-4-24 (Midnight) Progressive Web App

**Midnight** is a premium, real-time multiplayer, mobile-first Progressive Web App (PWA) built to play the authoritative **1-4-24 (Midnight)** dice game. It integrates a state-of-the-art **3D React Three Fiber (R3F) physics dice canvas** with a **retro-pixel video game aesthetic** featuring a soft CRT phosphor monitor glow.

---

## 🎮 Game Rules & Engine

Midnight is played with six six-sided dice, coordinated entirely by the server's authoritative state machine to prevent client-side manipulation.

1. **Standard Round**:
   * One player rolls at a time. The first player rolls all six dice.
   * **Mandatory Action**: After each roll, the active player must keep/set aside at least one die. Unkept dice are re-rolled.
   * **Qualification Phase**: To qualify for scoring, a player must set aside both a **1** and a **4** before the turn ends.
   * **Disqualification (Bust)**: If a player fails to keep both a **1** and a **4** once all six dice are set aside, they are disqualified (`BUST`) for that round.
   * **Score Calculation**: If qualified, the remaining four dice are totaled to determine the score (Maximum score is **24**).
   * **Win Condition**: The qualified player with the highest four-dice total wins.

2. **The Shootout (Tie-Breaker)**:
   * If two or more players tie at the highest score (or if everyone is disqualified at 0 points), they enter a **Shootout**.
   * Each shootout participant rolls all six dice in one go.
   * **Qualification rules are suspended**; the shootout score is the simple sum of all six dice (maximum 36).
   * Shootouts continue in a loop until the tie is broken.

## 🛠️ Architecture & Tech Stack

Midnight is designed as a decoupled monorepo managed via **npm workspaces**:

```
project_code/
├── client/          # React Frontend (Vite, TS, R3F, Cannon Physics, procedural Web Audio)
├── server/          # Node.js Backend (Express, TS, Socket.io, Drizzle ORM, Postgres migrator)
├── shared/          # Shared Typings (Unified network payloads & state shapes)
├── nginx/           # Reverse Proxy default routing mapping
└── docker-compose   # Container Orchestration mapping
```

### Key Technical Details
* **3D Physics Dice Throwing**: Powered by `@react-three/fiber` and `@react-three/cannon`. The dice are thrown authoritatively; when they settle, the client smoothly interpolates (lerps) the rotation to display the exact face value generated by the server.
* **Spectator Battery Saver**: The 3D physics canvas mounts *only* for the active player. Spectators receive a synchronized static scoreboard, saving CPU and battery.
* **8-Bit Procedural Audio**: Generates arcade-style bip clicks, roll clatters, Major arpeggio success pings, and sliding sawtooth DQ buzzers entirely procedurally via the **Web Audio API** (zero assets, 100% offline).
* **Decoupled CRT Theming**: Supports **Dark Mode** and **Light Mode** across CRT Green and CRT Amber phosphor styles, controlled from a centralized Theme Provider.
* **Automatic Migrations**: On server bootstrap, a Drizzle-compatible SQL initialization script runs automatically to verify and migrate all database tables (`users`, `matches`, `match_participants`).

## 🚀 How to Launch & Play (Quick Start)

The entire environment is fully containerized and orchestrated. Follow these steps to launch and test the game locally:

### 1. Prerequisite
Ensure you have **Docker** and **Docker Compose** installed on your machine.

### 2. Start the Environment
Navigate to the `project_code/` directory in your terminal and run:
```bash
docker compose up --build
```

This will automatically:
1. Spin up the **PostgreSQL** instance.
2. Build and boot the **Express server** (which will instantly auto-migrate the database tables).
3. Build and launch the **React client** (Vite dev environment).
4. Spin up the **Nginx reverse proxy** mapping port `80` to route traffic.

### 3. Open the Game in Your Browser
Open your browser and navigate to:
```
http://localhost
```

### 4. Multiplayer Sandbox Testing (Local)
To test the real-time multiplayer synchronization and shootout tie-breakers:
1. Open `http://localhost` in **Tab A**, enter a name (e.g. `ALICE`), and click **Host Match** to generate a 4-character room code (e.g. `K9W3`).
2. Open a **Private/Incognito Window** (Tab B), navigate to `http://localhost`, enter a different name (e.g. `BOB`), input the room code, and click **Enter Match Room**.
3. Tab A will instantly show Bob on the connected player roster!
4. Click **START MATCH** from the host tab (Tab A) to begin. Only the active player's screen will mount the 3D dice canvas. Watch the spectator scoreboard sync in real time!

## 🧪 Development & Testing Pipeline

We maintain a strict testing pipeline using **Vitest** to assert scoring, WebSockets, and database transactions:

### Run Tests Locally
1. Ensure dependencies are linked at the monorepo root:
   ```bash
   npm install
   ```
2. Execute the Vitest suite against the backend package:
   ```bash
   npm run test -w server
   ```

Our suite of **17 tests** covers:
* **Scoring Rules**: Validating 24-point maxima, missing `1` or `4` DQ rules, shootout sums.
* **Socket.io Handshakes**: Asserting player rolls authorization, index bounds validations, turn-keeping states.
* **Database Queries**: Verifying transaction-safe Drizzle inserts for matches, participants, and profile increments under game-over hooks.
